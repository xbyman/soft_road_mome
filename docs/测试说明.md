# 测试说明

## 1. 测试目的

本文档用于说明 `road_mome_demo_v1` 单机浏览器演示平台 V1.0 的测试范围、测试方法、测试项及测试结论，验证当前版本在本地演示场景下的可用性与异常处理能力。

本次测试重点关注以下目标：

- 验证页面可正常启动，主流程可正常使用。
- 验证演示样本加载、单帧检测、结果显示与导出功能正常。
- 验证关键异常输入和异常资源条件下，系统能够被校验模块拦截或给出可控提示。
- 验证页面不会因单个样本异常直接崩溃。

## 2. 测试环境

| 项目 | 说明 |
| --- | --- |
| 软件名称 | `road_mome_demo_v1` |
| 软件版本 | V1.0 |
| 系统形态 | 单机浏览器演示平台 |
| 运行模式 | 本地离线运行，`offline_mode = true` |
| 操作系统 | Windows 本地单机环境 |
| Python 环境 | Python 3.13.1 |
| 页面框架 | Dash 3.3.0 |
| 主要资源 | 本地 `config/`、`weights/`、`demo_data/`、`outputs/` |
| 演示样本 | `demo_data/index.json` 中 5 个样本 |

## 3. 测试范围

本次测试覆盖当前 V1.0 演示平台的以下内容：

- 启动阶段校验能力：`startup validation`
- 运行前样本校验能力：`runtime sample validation`
- 页面启动与样本下拉加载
- 5 个演示样本的正常运行
- “运行检测”功能
- “运行并导出”功能
- 四宫格结果显示
- `jpg / json / csv` 导出结果
- 异常路径下的错误拦截、状态提示与页面稳定性

本次测试不包含以下内容：

- 性能压测与定量性能指标评估
- 多用户并发访问测试
- 长时间连续稳定性测试
- 训练效果评估与模型精度报告
- 面向生产环境的部署验证

## 4. 测试方法

本次测试采用页面联调验证与校验逻辑验证相结合的方式进行，主要方法如下：

1. 在本地环境启动演示平台，检查页面是否可正常进入可操作状态。
2. 通过页面交互对样本下拉框、单帧运行、结果展示和导出流程进行正常路径验证。
3. 对 `demo_data`、权重文件、索引文件、样本字段和输出目录进行异常构造，验证启动校验与运行时样本校验是否能够正确拦截异常。
4. 检查状态区提示、页面显示和导出目录结果，确认异常出现时页面不直接崩溃，且错误范围可控。
5. 结合现有校验模块测试，对 `run_startup_validation`、`validate_index_file`、`validate_runtime_sample` 等关键异常分支进行验证。

## 6. 异常路径测试项

| 编号 | 异常场景 | 校验阶段 | 预期结果 | 实际结果 |
| --- | --- | --- | --- | --- |
| E-01 | `index.json` 为空 | 启动阶段 | 启动校验失败，页面不给出错误运行态 | 符合预期 |
| E-02 | 权重路径错误或权重缺失 | 启动阶段 | 启动校验失败，阻止进入推理流程 | 符合预期 |
| E-03 | `demo_data/jpg/` 缺失 | 启动阶段 | 启动校验失败，提示资源目录异常 | 符合预期 |
| E-04 | `demo_data/npz/` 缺失 | 启动阶段 | 启动校验失败，提示资源目录异常 | 符合预期 |
| E-05 | `index` 中缺 `jpg`、缺 `npz` 或路径错误 | 启动阶段 | 索引校验失败，阻止错误样本进入运行链路 | 符合预期 |
| E-06 | 样本字段 `shape` 异常 | 运行前样本校验 | 当前样本被拦截，不进入推理 | 符合预期 |
| E-07 | `meta[:,2]` 全 0 | 运行前样本校验 | 当前样本被拦截，不进入推理 | 符合预期 |
| E-08 | 输出目录不可写 | 启动阶段 | 输出目录校验失败，不进入不可控导出状态 | 符合预期 |

异常路径测试结果表明，当前版本已具备两级校验机制：

- `startup validation`：对配置、权重、`demo_data` 目录、`index.json`、输出目录进行启动前检查。
- `runtime sample validation`：对单帧样本关键字段、数组维度和有效 patch 条件进行运行前检查。

在上述异常场景下，系统表现为“异常可拦截、提示可理解、页面不直接崩溃”，符合演示平台的工程预期。

## 8. 结论

基于当前已完成的正常路径测试和异常路径测试，可以确认 `road_mome_demo_v1` V1.0 作为单机浏览器演示平台，已具备基本可用性。

当前版本的测试结论如下：

- 正常路径可用：页面启动、样本选择、单帧检测、四宫格显示及导出流程均已验证通过。
- 异常路径可控：针对索引、权重、样本结构、资源目录和输出目录等关键异常，系统能够通过启动校验或运行前样本校验进行拦截。
- 页面不崩：在单个样本异常或资源异常情况下，页面不会直接崩溃，能够维持可解释、可控制的错误提示状态。

同时应说明，当前版本仍定位为演示平台，而非生产系统。其测试结论主要证明现阶段功能链路完整、异常处理有效、页面稳定性满足演示要求，不代表已经完成性能、并发、长期稳定性及生产化部署层面的全面验证。
