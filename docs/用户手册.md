# 用户手册

## 1. 手册适用范围

本手册适用于 `road_mome_demo_v1`，即“Road-MoME 单机浏览器演示平台 V1.0”。

本文档面向普通使用者，说明本软件的基本使用方法，包括：

- 如何在本地启动软件
- 如何进入浏览器页面
- 如何选择样本并运行检测
- 如何执行结果导出
- 遇到常见异常时如何处理

本手册适用于当前单机离线演示版本，不适用于实时车载系统、在线服务系统或批处理系统。

## 2. 启动前准备

在启动软件前，请确认以下内容：

1. 已安装可用的 Python 运行环境。
2. 已在当前项目环境中安装运行依赖。
3. 项目目录完整，至少包含以下内容：
   - `app.py`
   - `config/`
   - `weights/`
   - `demo_data/`
   - `outputs/`
4. 演示数据目录存在以下文件或子目录：
   - `demo_data/index.json`
   - `demo_data/jpg/`
   - `demo_data/npz/`
5. 模型权重文件存在于 `weights/` 目录下。
6. `outputs/` 目录具有可写权限，用于保存导出结果和日志。

如果上述资源缺失，软件启动后可能无法进入正常运行状态，页面状态区会给出提示。

## 3. 软件启动方法

### 3.1 启动步骤

1. 打开命令行终端。
2. 进入项目根目录 `road_mome_demo_v1`。
3. 执行以下命令：

```bash
python app.py
```

### 3.2 页面访问方式

软件启动后，在本机浏览器中访问以下地址：

```text
http://127.0.0.1:8050
```

页面正常打开后，即可进入演示平台主界面。

### 3.3 启动后检查

进入页面后，建议先检查以下内容是否正常：

- 页面能够正常加载
- 页面中可看到权重路径和离线模式状态
- 样本选择下拉框中有可选样本
- 状态区未显示明显初始化失败信息

如果页面已打开但无法运行，请优先查看状态区中的初始化状态和错误提示。

## 4. 页面区域说明

当前页面主要由以下区域组成：

### 4.1 权重路径显示

用于显示当前加载的模型权重路径，便于确认当前使用的权重文件是否正确。

### 4.2 离线模式状态

用于显示当前系统是否以离线模式运行。当前版本为本地离线演示平台，一般显示为“已启用”。

### 4.3 样本选择下拉框

用于从 `demo_data/index.json` 提供的样本列表中选择一个演示样本。当前版本基于预置样本进行单帧演示。

### 4.4 阈值控件

用于设置当前运行时使用的检测阈值。使用者可根据演示需要调整阈值后再执行检测。

### 4.5 “运行检测”按钮

用于执行当前所选样本的单帧检测，并在页面中显示结果，但不执行文件导出。

### 4.6 “运行并导出”按钮

用于执行当前所选样本的单帧检测，并将结果同时导出到本地输出目录。

### 4.7 四宫格结果显示区

用于显示检测完成后的四宫格可视化结果，是页面中的主要结果展示区域。

### 4.8 统计信息区

用于显示当前样本对应的统计结果和最近一次导出路径等信息，便于查看运行结果摘要。

### 4.9 状态区

用于显示初始化状态、当前运行状态以及简要错误信息。出现问题时，应首先查看该区域。

## 5. 样本检测操作步骤

使用“运行检测”功能时，可按以下步骤操作：

1. 启动软件并在浏览器中打开页面。
2. 确认状态区未显示启动校验失败或初始化失败信息。
3. 在样本选择下拉框中选择一个演示样本。
4. 根据需要调整阈值控件。
5. 点击“运行检测”按钮。
6. 等待页面完成处理。
7. 在四宫格结果显示区查看可视化结果。
8. 在统计信息区查看当前帧的统计结果。
9. 在状态区确认本次运行是否成功。

如果运行成功，状态区会显示成功信息，四宫格结果区和统计信息区会同步更新。

## 6. 结果导出操作步骤

使用“运行并导出”功能时，可按以下步骤操作：

1. 启动软件并进入页面。
2. 在样本选择下拉框中选择一个演示样本。
3. 根据需要设置阈值。
4. 点击“运行并导出”按钮。
5. 等待页面完成检测和导出。
6. 在统计信息区查看最近一次导出路径。
7. 到本地输出目录中查看导出结果。

当前版本导出结果通常包括：

- 可视化结果图像
- 单帧结果 JSON 文件
- 结果摘要 CSV 文件

如导出成功，页面状态区会显示“运行并导出完成”或等效成功提示。

## 7. 常见问题与处理方法

### 7.1 页面可以打开，但没有可选样本

可能原因：

- `demo_data/index.json` 为空
- `index.json` 中样本配置不正确
- `demo_data/jpg/` 或 `demo_data/npz/` 缺失

处理方法：

1. 检查 `demo_data/index.json` 是否存在且内容非空。
2. 检查 `demo_data/jpg/` 和 `demo_data/npz/` 是否存在。
3. 检查索引中配置的样本路径是否正确。
4. 重新启动软件后再查看页面状态区。

### 7.2 点击“运行检测”后无法运行

可能原因：

- 未选择样本
- 启动校验未通过
- 当前样本数据异常

处理方法：

1. 确认已在下拉框中选择样本。
2. 查看状态区中的初始化状态和错误提示。
3. 检查当前样本对应的 `jpg` 和 `npz` 文件是否存在。
4. 如样本本身异常，请更换其他样本再次尝试。

### 7.3 页面提示权重路径错误或权重缺失

可能原因：

- 权重文件不存在
- 权重路径配置错误

处理方法：

1. 检查 `weights/` 目录下权重文件是否存在。
2. 检查相关配置中的权重路径是否正确。
3. 修正后重新启动软件。

### 7.4 页面提示样本校验失败

可能原因：

- 样本字段不完整
- 样本数组形状异常
- 样本中没有有效 patch

处理方法：

1. 更换其他样本重新尝试。
2. 检查当前样本数据文件是否完整。
3. 如为演示数据问题，修正数据后重新启动软件。

### 7.5 点击“运行并导出”后未生成文件

可能原因：

- 输出目录不可写
- 导出过程被异常中断

处理方法：

1. 检查 `outputs/` 目录是否存在。
2. 检查当前用户是否具有写入权限。
3. 查看状态区是否有导出失败提示。
4. 必要时查看日志文件后重新尝试。

### 7.6 页面显示异常信息，但没有直接崩溃

说明系统已对异常进行了拦截。此时建议：

1. 先阅读状态区中的错误提示。
2. 按提示检查权重、样本、索引或输出目录。
3. 修正问题后重新启动软件。

## 8. 注意事项

- 本软件为单机离线演示平台，使用前应保证本地资源完整。
- 当前版本仅支持基于 `demo_data/index.json` 的预置样本演示。
- 当前版本支持单帧检测，不支持实时视频流处理。
- 当前版本支持运行和导出，不支持批量处理。
- 页面异常时，不要反复点击按钮，建议先查看状态区提示。
- 如果导出结果未生成，应优先检查输出目录和写入权限。
- 如需判断软件是否已正常初始化，应以状态区信息为准。
- 本手册面向使用操作，不涉及模型训练、模型修改或系统二次开发内容。
